01

UI 개발 가이드

토큰을 단일 소스로, 프리미티브 컴포넌트를 쌓고, 페이지는 조합만 한다. 하드코딩 금지.

폴더 구조

src/
  styles/
    index.css        # @theme 토큰 단일 소스 (design-tokens.json에서 이관)
  ui/               # 프리미티브 (디자인 시스템 그대로)
    Button.tsx  Input.tsx  Card.tsx  Stat.tsx  Badge.tsx
    Progress.tsx  Alert.tsx  Modal.tsx  Skeleton.tsx  Icon.tsx
  components/       # 기능 조합 (도메인)
    TodayVerse.tsx  ContributionGraph.tsx  Sidebar.tsx ...
  pages/            # 화면 (ui + components 조합만)

규약

항목규칙
토큰styles/index.css의 @theme가 유일 소스. 값은 항상 var(--...). hex 하드코딩 금지
CSS 방법컴포넌트 클래스 + BEM-lite modifier(.btn--primary). 유틸(Tailwind)은 레이아웃·간격에 한정
네이밍컴포넌트 PascalCase / CSS kebab-case / 토큰 --group-name
숫자모든 통계·수치는 font-variant-numeric: tabular-nums
상태모든 인터랙티브 요소에 hover · active · :focus-visible · [disabled] 정의 필수
반응형모바일 우선(min-width). 브레이크포인트 640 / 768 / 1024 / 1280
모션prefers-reduced-motion 전역 존중. 오케스트레이션은 잔디·저장 체크 1곳
브라우저Chrome/Edge/Safari 최신 2버전, iOS 15+, Android Chrome

Do / Don't

Do

  • 파랑은 CTA·링크·활성·잔디에만
  • 카드는 --r-xl(16) + 1px 보더, 그림자 최소
  • 숫자는 tabular, 라벨은 sentence case
  • 포커스 링 항상 노출(:focus-visible)

Don't

  • 파랑 그라디언트를 일반 배경에 남발
  • 이모지를 아이콘으로 사용(🇰🇷📷🔥)
  • 페이지마다 :root 재정의
  • <div onClick>로 버튼 흉내(→ <button>)

02

마크업 명세

의미에 맞는 태그, 접근 가능한 구조. 시각이 아니라 역할로 마크업한다.

시맨틱 태그 규약

역할태그주의
페이지 머리<header>로고·상단바
내비게이션<nav aria-label>사이드바/탭바
본문<main>페이지당 1개
구획<section> / <article>제목(h2/h3) 동반
목록(서재·기록)<ul><li>나열은 리스트로
동작<button>같은 화면 액션
이동<a href>라우팅
입력<label>+<input>label for 연결 필수
이미지<img alt>의미 있으면 alt, 장식은 alt=""

컴포넌트 마크업 예시

<!-- Field: label 연결 + 오류 접근성 -->
<div class="field">
  <label for="verse-range">절 범위</label>
  <input id="verse-range" class="input" aria-invalid="true"
         aria-describedby="vr-help">
  <p id="vr-help" class="help err">1 이상을 입력해 주세요</p>
</div>

<!-- Alert / Modal 접근성 -->
<div class="alert alert--error" role="alert"> … </div>
<div role="dialog" aria-modal="true" aria-labelledby="m-title"> … </div>

<!-- Jandi: 스크린리더용 요약 -->
<div class="jandi" role="img"
     aria-label="최근 1년 필사 잔디, 256일 기록">…</div>
ARIA 원칙 — 네이티브 태그로 되면 ARIA를 쓰지 않는다. role=dialog엔 포커스 트랩 + Esc 닫기, 로딩엔 aria-busy="true", 토스트는 role="status"(성공)/role="alert"(오류).

03

웹 접근성(WA) 체크리스트

KWCAG 2.1 / WCAG 2.1 AA 기준. 상태는 현재 코드베이스 실측 기준.

현 코드 실측: :focus-visible 0곳 · <img> alt 0/2 · 대비 미검증 — 아래 예정 항목이 다수. P1 개선 대상.
원칙검사 항목적용 방법상태
인식대체 텍스트의미 이미지 alt, 아이콘 버튼 aria-label예정
인식명료성(대비)본문 4.5:1, 큰 텍스트 3:1. ink/body on 흰 배경 통과부분
인식색만으로 정보전달 금지상태는 색+아이콘/텍스트 병기(잔디는 aria-label)부분
운용키보드 접근모든 인터랙티브 <button>/<a>, Tab 순서 논리적부분
운용초점 이동·가시성:focus-visible 2px 링(전역 정의) · 모달 포커스 트랩예정
운용충분한 타깃 크기버튼·탭 최소 44×44 (모바일)부분
운용반응형·확대200% 확대·모바일 무횡스크롤(검증됨)적용
운용움직임 최소화prefers-reduced-motion 존중부분
이해레이블 제공label for 연결, placeholder만 사용 금지예정
이해오류 정정 안내aria-invalid + aria-describedby로 원인·해결 제시예정
이해일관성같은 액션 = 같은 이름(오늘 필사 시작 → "저장됨")부분
견고문법 준수유효한 HTML, 중복 id 없음부분
견고이름·역할·값커스텀 UI에 적절한 role/aria-*예정
견고언어 지정<html lang="ko">부분

04

공통 레이아웃 · 템플릿

한 App Shell, 다섯 페이지 템플릿. 모바일=하단 탭바, 데스크탑=좌측 사이드바.

App Shell

<!-- 모바일: header(sticky) + main + tabbar(fixed bottom) -->
<!-- 데스크탑(≥1024): aside(sidebar sticky) + main(max-w 1120, 중앙) -->
<div class="app-shell">
  <aside class="side-nav">…</aside>   // ≥1024만
  <main class="content">…</main>
  <nav class="tab-bar">…</nav>     // <1024만
</div>
토큰값
content max-width1120px (중앙 정렬)
content padding모바일 16px / 데스크탑 24~32px
sidebar224px 고정, sticky
tab-bar 높이56px + safe-area
섹션 간격24px(mobile) / 32px(desktop)

페이지 템플릿

템플릿화면구성반응형

구축 절차 (플러그인)

  1. Tokens Studio 플러그인 → docs/design-tokens.json Import → Create Variables/Styles로 색·타입·간격·라운드·그림자 자동 생성.
  2. html.to.design 플러그인 → 디자인 시스템 문서 URL / 화면 Artifact URL을 Import → 편집 가능한 프레임으로 변환.
  3. 임포트된 요소를 위 네이밍으로 Component + Variant화. 스타일은 1단계 Variables로 연결.
실제 .fig 생성은 사람/플러그인 단계입니다. 위 3종 산출물이 그 재료이며, 네이밍을 코드 클래스와 맞추면 §7 핸드오프가 무손실로 이어집니다.

07

개발 핸드오프

Figma → 코드 무손실. 토큰·컴포넌트·redline·상태를 규약으로 고정.

토큰 이관

/* styles/index.css — design-tokens.json에서 이관 */
@theme {
  --color-blue-500: #3B82F6;   --color-ink: #17324A;
  --radius-xl: 16px;         --shadow-md: 0 6px 18px rgba(23,50,74,.08);
  /* … 전체는 design-tokens.json 참조 … */
}

Redline 규약

  • 간격·크기는 토큰명으로 표기 — "16px"이 아니라 space/4, radius/xl
  • 컴포넌트는 variant/state로 지정 — "파란 버튼"이 아니라 Button/primary/md
  • 각 상태(hover·focus·disabled·loading·error)의 스펙을 함께 전달
  • 인터랙션 노트: 전환 150ms standard, 오케스트레이션(잔디·저장 체크)만 별도 명시

Dev Mode MCP 루프 (Figma가 서면)

단계도구결과
토큰 추출get_variable_defsFigma 변수 → 코드 토큰 대조·동기화
코드 생성get_code (선택 프레임)프리미티브 클래스로 매핑한 React/CSS 초안
스냅샷get_image구현 대조용 레퍼런스

컴포넌트 Definition of Done

  • 모든 variant·state 구현 + :focus-visible 링
  • 키보드 조작 가능 · 스크린리더 레이블
  • tabular 숫자 · reduced-motion 대응
  • 토큰만 사용(하드코딩 0) · 모바일~데스크탑 무횡스크롤
